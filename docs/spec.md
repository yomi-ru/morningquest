# Morning Quest 仕様書

## 1. 概要

Morning Quest は、寮の 6 人ユニットを対象に、朝の起床確認と外出準備を支援するモバイルアプリです。7:00 の起床アラーム、共有スペース QR による起床確認、未起床メンバー通知、天気・予定・持ち物チェック、8:50 の在寮アラームを通じて、遅刻や寝坊を減らします。

## 2. 目的

- 寮の共有スペースまで移動しないと起床確認できない設計にして、二度寝を防ぐ。
- 6 人ユニットの中で未起床者がいる場合に早く気づけるようにする。
- 朝の出発前に天気、予定、持ち物をひと目で確認できるようにする。
- 出発予定時刻を過ぎても寮にいる場合に、追加アラームで外出を促す。

## 3. 想定ユーザー

- 寮に住む学生
- 6 人前後のユニット単位で生活しているユーザー
- Google Calendar で予定を管理しているユーザー

## 4. MVP の主要機能

### 4.1 7:00 起床アラーム

- 毎朝 7:00 にアラームを鳴らす。
- 音は急に大音量で鳴らすのではなく、段階的に覚醒を促す設計にする。
- MVP では「短い明るいチャイム」「段階的な音量上昇」「繰り返しリズム」を採用する。
- 本実装ではユーザーの端末設定、マナーモード、OS のバックグラウンド制限を考慮する。

### 4.2 QR 起床確認

- 寮のユニット共有スペースに設置された QR コードを読み取ると起床確認が完了する。
- QR はユニットごとに発行する。
- 不正防止のため、本実装では QR の有効期限、日付署名、位置情報の補助判定を検討する。
- MVP プロトタイプでは「QR 読み取り完了」ボタンで代替する。

### 4.3 5 分以内のユニット未起床通知

- 7:00 のアラーム開始から 5 分以内に、6 人ユニットのうち 1 人でも起床確認が未完了なら、ユニット内の起床済みメンバーまたは全員に通知する。
- 通知内容の例: 「ユニット内に未起床メンバーがいます。共有スペースで確認してください。」
- 通知の乱発を避けるため、同一朝の通知は原則 1 回に制限する。
- 本実装ではプッシュ通知基盤が必要。

### 4.4 ホーム画面

ホーム画面では以下を表示する。

- 今日の天気
- 今日の予定
- 持ち物リスト
- ユニットメンバーの起床ステータス
- 現在のクエスト状態

今日の天気は Web ページへのアクセス時に Open-Meteo Forecast API へ自動接続し、徳島県名西郡神山町の緯度経度で取得する。プロトタイプでは `daily=weather_code,precipitation_probability_max` と `timezone=Asia/Tokyo` を指定し、本日の天気コードと最大降水確率を表示する。最大降水確率が 70% 以上の場合は、持ち物リストに「傘」を自動追加する。

今日の予定は Google Calendar の公開 URL から取得する。プロトタイプでは iCal 形式の公開 URL、`webcal://` URL、または `https://calendar.google.com/calendar/u/0?cid=...` 形式の画面 URL を入力して連携し、直近の予定をホームに表示する。`cid` 形式の画面 URL は公開 iCal URL に変換して取得を試みる。連携後は「本日の予定を展開」ボタンで今日の予定一覧を開閉する。本実装では公開 URL だけでなく、プライベート予定を扱う場合に Google OAuth と Calendar API の導入を検討する。

### 4.5 8:50 在寮アラーム

- 8:50 時点でユーザーが徳島県名西郡神山町神領字西上角175-1 付近にいると判定された場合、外出を促すアラームを鳴らす。
- プロトタイプでは基準座標を `33.972804,134.362465` とし、半径 75m 以内を「寮付近」と判定する。
- 本実装では端末の位置情報権限が許可済みである前提で、バックグラウンドまたは 8:50 の起動処理内で自動的に位置情報を確認する。
- 位置情報が取得できない場合に備えて、手動の「外出済み」ボタンや寮 Wi-Fi 接続状態の補助判定を検討する。
- ブラウザプロトタイプでは 8:50 チェック時に Geolocation API を自動実行する。ブラウザや実行元によっては HTTPS または localhost が必要になる。

## 5. 画面仕様

### 5.1 ホーム

- 上部に今日の日付、天気、次の予定を表示する。
- 中央に朝のクエストカードを表示する。
- 下部に持ち物チェックリストを表示する。
- ユニットメンバーの起床状況を一覧表示する。

### 5.2 起床確認

- アラーム中であることを表示する。
- 「共有スペース QR を読み取る」導線を表示する。
- QR 読み取り完了後は自分のステータスを「起床済み」にする。

### 5.3 通知

- 未起床メンバーがいる場合、通知カードを表示する。
- 本実装では OS のプッシュ通知に置き換える。

## 6. データモデル案

### User

- `id`: ユーザー ID
- `name`: 表示名
- `unitId`: 所属ユニット ID
- `wakeStatus`: 当日の起床状態
- `calendarConnected`: Google Calendar 連携状態

### Unit

- `id`: ユニット ID
- `name`: ユニット名
- `members`: メンバー ID 一覧
- `qrSecret`: QR 署名用シークレット

### WakeLog

- `id`: ログ ID
- `userId`: ユーザー ID
- `unitId`: ユニット ID
- `date`: 対象日
- `confirmedAt`: 起床確認時刻
- `method`: `qr`

### ChecklistItem

- `id`: アイテム ID
- `userId`: ユーザー ID
- `label`: 持ち物名
- `checked`: チェック状態
- `daysOfWeek`: 表示曜日

## 7. 通知ルール案

- 7:00: 起床アラームを開始。
- 7:05: ユニット内に未起床者が 1 人以上いる場合に通知。
- 8:50: 寮にいる場合に出発アラームを開始。
- 8:50 アラームはユーザーが「出発した」を押す、またはジオフェンス外に出たら停止する。

## 8. 外部連携

- Open-Meteo Forecast API: 徳島県名西郡神山町の本日の天気と最大降水確率を取得。リクエストは `https://api.open-meteo.com/v1/forecast` を利用し、神山町のプロトタイプ座標は緯度 `33.986798`、経度 `134.397042` とする。最大降水確率が 70% 以上の場合、持ち物リストへ「傘」を自動追加する。
- Google Calendar 公開 URL: iCal 形式の公開 URL から今日の予定を取得。`cid` 付きの Google Calendar 画面 URL が入力された場合は、`cid` をカレンダー ID として公開 iCal URL に変換する。公開カレンダーでない場合、公開 URL では詳細を取得できない。ブラウザ直 fetch は CORS 制限を受ける可能性があるため、プロトタイプではローカルプロキシ、本実装ではサーバー側プロキシまたは Google Calendar API を推奨する。
- Push Notification: 未起床通知と在寮通知。
- QR Scanner: 共有スペース QR 読み取り。
- Location / Geofence: 8:50 の在寮判定。
- Geolocation API: プロトタイプでは 8:50 チェック時に自動で現在地を取得し、神山町神領字西上角175-1 付近から半径 75m 以内かを判定する。範囲内ならアラームを鳴らす。

## 9. 非機能要件

- 個人の位置情報や起床状況はユニット内で必要最小限だけ共有する。
- 位置情報の利用目的を明示し、ユーザーの同意を取得する。
- QR のスクリーンショット共有などの不正対策を検討する。
- アラームや通知は OS 制限の影響を受けるため、iOS/Android で個別検証する。

## 10. 今後の検討事項

- 未起床通知を誰に送るか: 全員、起床済みの人のみ、ユニットリーダーのみ。
- 起床確認の締切を 5 分固定にするか、ユニットごとに設定可能にするか。
- 寮から出た判定を位置情報だけに頼るか、Wi-Fi や手動ボタンも組み合わせるか。
- アラーム音をどの程度カスタマイズできるようにするか。
- ゲーム要素として連続起床日数、ユニット達成率、称号などを入れるか。
