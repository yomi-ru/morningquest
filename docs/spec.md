# Morning Quest 仕様書

## 1. 文書情報

- 文書名: Morning Quest 仕様書
- 対象プロダクト: Morning Quest
- 文書目的: プロトタイプ実装と本実装検討の基準となる、機能仕様・技術仕様・受け入れ条件を定義する
- 想定読者: プロダクトオーナー、デザイナー、フロントエンド実装者、バックエンド実装者、QA
- 対象バージョン: Prototype v1

## 2. 背景と目的

Morning Quest は、寮生活における寝坊防止と朝の出発準備を支援するアプリである。単にアラームを鳴らすだけではなく、共有スペースに移動して QR を読み取る行動を起床完了条件にすることで、二度寝の発生を減らすことを狙う。

本プロトタイプでは、以下を主要価値として提供する。

- 7:00 の起床アラーム
- 共有スペース QR による起床確認
- 7:05 の未起床メンバー検知
- ホーム画面での天気、予定、持ち物確認
- 8:50 以降の在寮アラーム
- 実時刻連動の警告表示

## 3. 対象ユーザー

- 寮に住む学生
- 6 人前後のユニット単位で生活するユーザー
- 朝の予定を Google Calendar で管理しているユーザー
- スマートフォンで位置情報とカメラを使えるユーザー

## 4. スコープ

### 4.1 本仕様に含むもの

- ブラウザベースのプロトタイプ UI
- 起床アラームの疑似再生
- QR 読み取りによる起床判定
- 天気情報の自動取得
- Google Calendar 公開 URL からの予定取得
- Geolocation API による在寮判定
- 実時刻に同期した Prototype Clock 表示

### 4.2 本仕様に含まないもの

- ネイティブアプリとしてのバックグラウンド起動保証
- 本番用の認証基盤
- 本番用のプッシュ通知基盤
- サーバーサイド永続化
- ユーザー登録、寮管理者画面、運用管理画面
- QR の日次署名や改ざん防止の本番実装

## 5. 用語定義

- ユニット: 寮内で生活を共にする 6 人前後の居住グループ
- 起床確認: 共有スペースに設置された起床用 QR の読取をもって完了する当日朝の確認行為
- 在寮判定: 特定座標から半径 75m 以内に存在する状態
- Prototype Clock: ホーム上部に表示される現在時刻表示コンポーネント
- 起床用 QR: `https://yomiru.net/morningquest/wakeup` を格納した QR

## 6. 前提条件

### 6.1 利用環境

- 対応形態はブラウザプロトタイプとする
- 位置情報利用時は `https://` または `http://localhost` で配信されていること
- QR カメラ読取にはカメラ権限が必要
- Google Calendar 連携は公開 iCal ベースで行う

- UI は静的 HTML/CSS/JavaScript で構成する

- Google Calendar 取得はローカルプロキシ経由を優先する
- データはインメモリで保持し、ページリロードで初期化される

## 8. 機能仕様

### 8.1 7:00 起床アラーム

#### 8.1.1 概要

ユーザーに朝の起床を促す疑似アラームを再生する。

#### 8.1.2 挙動

- `7:00 アラームを鳴らす` ボタン押下で起床アラームを開始する
- 音は短いチャイムを反復し、段階的に音量を上げる
- クエストステータスを `アラーム中` に変更する
- クエストメッセージを「共有スペースに移動して QR を読み取る」内容に変更する
- 起床用 QR 読取成功時にアラームを停止する

#### 8.1.3 制約

- ブラウザのオーディオ再生制限により、ユーザー操作後にのみ音が鳴る
- OS バックグラウンド制御は考慮対象外とする

### 8.2 QR 起床確認

#### 8.2.1 概要

共有スペースで起床用 QR を読み取り、当日の起床確認を完了する。

#### 8.2.2 起床完了条件

- QR の読取結果文字列を `trim()` する
- 次の文字列と完全一致した場合のみ起床確認成功とする

`https://yomiru.net/morningquest/wakeup`

#### 8.2.3 UI 要件

- 起床カードに `QR を読み取る` ボタンを表示する
- 読取パネルには以下を表示する
  - タイトル
  - カメラ読取領域
  - ステータスメッセージ
  - `写真から読み取る` ボタン
  - `再試行` ボタン
  - `閉じる` ボタン

#### 8.2.4 カメラ読取挙動

- `QR を読み取る` 押下で読取パネルを開く
- `html5-qrcode` の `start()` でカメラを起動する
- 背面カメラを優先するため `facingMode: "environment"` を指定する
- 読取成功時は起床完了処理を実行する
- 不一致時は起床扱いせず、不一致メッセージを表示する
- `閉じる`、読取成功、画面非表示、ページ離脱時にスキャナを停止する

#### 8.2.5 写真読取フォールバック

- カメラ読取が難しい端末向けに `写真から読み取る` を提供する
- 画像ファイル選択後、`html5-qrcode` の `scanFile()` で QR を判定する
- 判定ロジックはカメラ読取時と同じ完全一致判定とする

#### 8.2.6 成功時処理

- 自ユーザーの `awake` 状態を `true` にする
- クエストバッジを `起床済み` にする
- クエスト文言を「共有スペース QR の読み取りが完了しました。今日の朝クエストは順調です。」にする
- アラーム音を停止する
- メンバー一覧を再描画する
- 通知カードを非表示にする
- 読取パネルを閉じる

#### 8.2.7 エラー時処理

- ライブラリ未読込: QR ライブラリ読込失敗メッセージを表示
- カメラ起動失敗: 権限、HTTPS、端末設定の確認を促す
- 読取不一致: 起床用 QR ではないことを表示
- 画像読取失敗: 別画像での再試行を促す

### 8.3 7:05 未起床通知

#### 8.3.1 概要

7:00 の起床アラーム開始から 5 分経過時点で未起床者が残っているかを確認する。

#### 8.3.2 挙動

- `7:05 未起床チェック` ボタン押下で未起床者数を算出する
- 全員起床済みなら成功メッセージを表示する
- 未起床者が 1 人以上なら人数と氏名を通知カードに表示する

#### 8.3.3 本番拡張想定

- プッシュ通知送信
- 通知先制御
- 同日重複通知防止

### 8.4 ホーム画面

#### 8.4.1 表示項目

- Prototype Clock
- 天気
- 次の予定
- 出発目安
- 起床クエストカード
- ユニットメンバー起床状況
- 持ち物リスト

### 8.5 天気取得

#### 8.5.1 データソース

- API: Open-Meteo Forecast API
- エンドポイント: `https://api.open-meteo.com/v1/forecast`

#### 8.5.2 リクエスト仕様

- `latitude=33.986798`
- `longitude=134.397042`
- `daily=weather_code,precipitation_probability_max`
- `timezone=Asia/Tokyo`
- `forecast_days=1`

#### 8.5.3 挙動

- ページ表示時に自動取得する
- `天気を再取得` ボタンで手動再取得できる
- `weather_code` を日本語ラベルへ変換して表示する
- `precipitation_probability_max >= 70` の場合、持ち物リストへ `傘` を自動追加する
- 70% 未満の場合、自動追加された傘のみ削除対象とする

#### 8.5.4 エラー時処理

- HTTP エラー時は天気取得失敗メッセージを表示する
- 必須フィールド欠落時はレスポンス異常として扱う

### 8.6 Google Calendar 連携

#### 8.6.1 概要

Google Calendar の公開 URL から本日の予定を取得し、直近予定と予定一覧を表示する。

#### 8.6.2 受付 URL 形式

- iCal 公開 URL
- `webcal://` URL
- `https://calendar.google.com/calendar/u/0?cid=...` 形式の画面 URL

#### 8.6.3 URL 正規化

- `webcal://` は `https://` に置換する
- `cid` 付き URL は `cid` をデコードし、公開 iCal URL へ変換する
- 変換先形式:
  - `https://calendar.google.com/calendar/ical/{calendarId}/public/basic.ics`

#### 8.6.4 取得方式

- `http://localhost` または `https://` 配信時は `/api/calendar` ローカルプロキシ経由を優先する
- プロキシ失敗時は直接 `fetch()` を試行する
- 直接取得は CORS により失敗する可能性がある

#### 8.6.5 iCal 解析仕様

- `VEVENT` を抽出する
- `SUMMARY`, `DTSTART`, `DTEND`, `LOCATION`, `RRULE` を解析対象とする
- 当日予定に加え、`DAILY`, `WEEKLY` の基本的な繰り返し予定を当日分へ展開する
- 直近予定をサマリ表示する
- `本日の予定を展開` ボタンで今日の予定一覧を開閉する

#### 8.6.6 エラー時処理

- URL 未入力
- iCal 形式ではないレスポンス
- HTTP エラー
- CORS 失敗
- 公開設定不足

上記はすべて UI 上の連携失敗メッセージとして表示する。

### 8.7 持ち物リスト

#### 8.7.1 初期項目

- 学生証
- ノート PC
- 水筒
- イヤホン

#### 8.7.2 動的項目

- 天気条件により `傘` を自動追加する
- 各項目はチェックボックスで ON/OFF を切り替える
- 完了件数をバッジ表示する

### 8.8 在寮判定と 8:50 アラーム

#### 8.8.1 判定基準

- 基準地点: 徳島県名西郡神山町神領字西上角175-1
- 座標: 緯度 `33.972804`, 経度 `134.362465`
- 判定半径: 75m

#### 8.8.2 判定方式

- Geolocation API で現在地を取得する
- ハーサイン式で基準地点との距離を算出する
- 75m 以内なら `inDorm = true` とする

#### 8.8.3 8:50 チェック挙動

- `8:50 チェック` 押下時に位置情報を自動取得する
- 寮付近にいる場合は出発アラーム音を再生する
- 寮付近外なら追加アラームは鳴らさない
- 位置情報取得失敗時は失敗メッセージを表示する

### 8.9 Prototype Clock

#### 8.9.1 基本挙動

- 端末の現在時刻を `ja-JP` の `HH:MM` 形式で表示する
- 1 秒間隔で更新する
- 外部スクリプト失敗時に備え、HTML 内フォールバック時計を持つ

#### 8.9.2 警告表示

- 時間帯 `8:50〜12:15` の間に `inDorm = true` の場合、背景色を赤にする
- 時間帯外、または寮付近外では通常色を表示する
- 位置情報は 60 秒間隔で再評価する

## 11. データモデル

### 11.1 User

- `id`: ユーザー ID
- `name`: 表示名
- `unitId`: 所属ユニット ID
- `wakeStatus`: `awake | sleeping`
- `calendarConnected`: カレンダー連携状態

### 11.2 Unit

- `id`: ユニット ID
- `name`: ユニット名
- `members`: メンバー ID 一覧
- `qrSecret`: 将来の署名付き QR 用シークレット

### 11.3 WakeLog

- `id`: ログ ID
- `userId`: ユーザー ID
- `unitId`: ユニット ID
- `date`: 対象日
- `confirmedAt`: 起床確認時刻
- `method`: `qr`
- `value`: 読み取った QR 値

### 11.4 ChecklistItem

- `id`: アイテム ID
- `userId`: ユーザー ID
- `label`: 項目名
- `checked`: チェック状態
- `autoAddedByWeather`: 天気由来の自動追加かどうか

## 12. 外部インターフェース仕様

### 12.1 Open-Meteo

- Method: `GET`
- URL: `https://api.open-meteo.com/v1/forecast`
- 主な利用レスポンス
  - `daily.weather_code[0]`
  - `daily.precipitation_probability_max[0]`

### 12.2 Google Calendar

- 入力値: 公開 URL
- ローカルプロキシ: `GET /api/calendar?url=...`
- 返却 Content-Type 想定: `text/calendar`

### 12.3 Geolocation API

- `navigator.geolocation.getCurrentPosition()`
- `enableHighAccuracy: true`
- `maximumAge: 60000`
- `timeout: 10000`

### 12.4 QR Reader

- CDN: `https://unpkg.com/html5-qrcode@2.3.8/html5-qrcode.min.js`
- カメラ読取: `start()`
- 画像読取: `scanFile()`
- 停止: `stop()`, `clear()`

## 13. エラーハンドリング

### 13.1 共通方針

- 例外で画面全体を停止させず、対象機能のメッセージ領域に失敗理由を表示する
- ユーザーが次の行動を取れる文言にする
- 状態更新に失敗した場合、既存状態を保持する

### 13.2 機能別エラー

- 天気
  - API 応答なし
  - 必須フィールド欠落
- カレンダー
  - 無効 URL
  - 公開設定不足
  - iCal 形式でないレスポンス
  - CORS / プロキシ失敗
- QR
  - カメラ権限拒否
  - HTTPS 条件未達
  - ライブラリ未読込
  - 画像読取失敗
  - URL 不一致
- 位置情報
  - 権限拒否
  - タイムアウト
  - 端末設定無効

## 14. 非機能要件

### 14.1 パフォーマンス

- Prototype Clock 更新は 1 秒間隔とする
- 在寮判定再評価は 60 秒間隔とする
- QR 読取停止時は必ずカメラリソースを解放する

### 14.2 セキュリティとプライバシー

- 位置情報の利用目的を画面上で明示する
- 起床状態はユニット内共有前提だが、必要最小限に限定する
- 公開 iCal URL は読み取り専用だが秘匿情報として扱う
- QR 値は本番では署名付きや時限付きへの拡張を前提とする

### 14.3 互換性

- カメラ読取はモバイルブラウザ優先
- iPhone Chrome では写真読取フォールバックを許容
- 位置情報は HTTPS または localhost でのみ動作保証

## 15. 受け入れ条件

### 15.1 起床確認

- `QR を読み取る` 押下で読取パネルが開く
- カメラ読取または画像読取で起床用 QR を判定できる
- `https://yomiru.net/morningquest/wakeup` と完全一致した場合のみ起床済みになる
- 別 URL の QR は起床済みにならない

### 15.2 天気

- ページ表示時に自動取得される
- 降水確率 70% 以上で `傘` が追加される

### 15.3 カレンダー

- 公開 URL を入力して連携すると直近予定が表示される
- 本日の予定一覧を展開できる

### 15.4 在寮判定

- 8:50 チェックで現在地判定が実行される
- 寮付近ならアラームが鳴る
- 寮付近外なら鳴らない

### 15.5 Prototype Clock

- 現在時刻が表示される
- 8:50〜12:15 かつ寮付近で背景が赤になる
